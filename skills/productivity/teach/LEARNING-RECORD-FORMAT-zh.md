# 学习记录格式

学习记录位于 `./learning-records/`，使用顺序编号：`0001-slug.md`、`0002-slug.md` 等。懒加载创建目录——只有在第一个记录被写入时才创建。

它们是教学中的 ADR 等效物：它们捕获非显而易见的课程、关键洞察和已陈明的先前知识，这些将指导未来的 sessions。它们用于计算最近发展区。

## 模板

```md
# {学到或确定的内容的简短标题}

{1-3 句话：学到了什么（或确定了什么先前知识），以及为什么它对未来的 sessions 很重要。}
```

这就是整个格式。一个学习记录可以是一个段落。价值在于记录_现在知道这个_以及_它为什么改变下一步教什么_——而不是填写 sections。

## 可选 Sections

仅当它们增加真正的价值时才包含。大多数记录不需要它们。

- **Status** frontmatter （`active | superseded by LR-NNNN`）——当早期的理解被证明是错误的并被替换时有用。
- **Evidence** ——用户如何展示理解（回答的问题、完成的练习、引用的先前经验）。当声明可能被重新审视时有用。
- **Implications** ——这对未来的 sessions 开启或排除了什么。当不显而易见时值得记录。

## 编号

扫描 `./learning-records/` 获取最高的现有编号并加一。

## 何时编写学习记录

当以下任何一个为真时编写一个：

1. **用户展示了对某非琐碎事物的真正理解**——不仅仅是接触过，而是他们能正确使用该概念的证据。这为下一步教什么设置了新基线。
2. **用户披露了先前知识**——"我已经知道 X。"记录它，以便未来的 sessions 不会重新教它。也记录声明的_深度_。
3. **误解被纠正**——用户之前相信错误的东西，现在明白为什么。这些是高价值的：它们为相关主题预测未来的绊脚石。
4. **Mission 响应学习而转变**——用户发现自己关心的是与他们想到的不同的事情。交叉链接到 [[MISSION-zh.md]] 并更新它。

### 什么_不_符合

- 仅仅被覆盖的材料。覆盖不是学习。等待证据。
- 已经作为术语定义简洁地捕获在 [[GLOSSARY-zh.md]] 中的任何内容。不要重复。
- 逐 session 的活动日志。学习记录不是日记——它们是决策级别的洞察。

## 替代

当稍后的记录与早期的记录相矛盾时（用户的理解加深或得到纠正），将旧记录标记为 `Status: superseded by LR-NNNN` 而不是删除它。理解如何演化的历史本身就是有用的信号。
